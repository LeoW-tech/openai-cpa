# openai-cpa 常用命令

这份文档专门存放本项目的常用命令，所有命令都已经统一整理为“带项目运行目录、整行可复制执行”的单行命令。

默认项目目录固定为：

```text
/Users/meilinwang/Projects/openai-cpa-Public
```

当前本机日常维护默认以根目录 `scripts/` 下的脚本和本地容器 `openai-cpa-local` 为准。
README 里的 `docker-compose.yml` 更接近上游示例，不是这里日常在 `main` 分支维护本机服务的主入口。

## 高频命令

这两条是日常最常用的命令，建议优先看这里。

### 1. `main` 分支一键快速重启当前容器

适用场景：只是想切回 `main` 并快速重启当前正在运行的本地容器，例如改了 `data/config.yaml`、容器状态不对，或者想确认现在跑的是 `main` 分支下现有镜像。
这条命令不会重新构建镜像，也不会把你刚 `git pull` 下来的新代码装进容器里。

推荐入口：

```bash
cd /Users/meilinwang/Projects/openai-cpa-Public && git checkout main && ./scripts/restart_local_container.sh
```

兜底原生命令：

```bash
cd /Users/meilinwang/Projects/openai-cpa-Public && git checkout main && docker restart openai-cpa-local
```

### 2. `main` 分支一键拉最新代码后重建镜像并重新启动

适用场景：你想确保容器实际运行的是 `origin/main` 最新代码，或者你已经改了项目代码、依赖、镜像构建内容，需要重建镜像再启动。
`./scripts/rebuild_local_container.sh` 会自动备份当前 `data/config.yaml`。

推荐入口：

```bash
cd /Users/meilinwang/Projects/openai-cpa-Public && git checkout main && git pull --ff-only origin main && ./scripts/rebuild_local_container.sh
```

兜底原生命令：

```bash
cd /Users/meilinwang/Projects/openai-cpa-Public && git checkout main && git pull --ff-only origin main && mkdir -p data && [ -f data/config.yaml ] && cp data/config.yaml data/config.yaml.bak-$(date +%Y%m%d-%H%M%S) || true && (docker rm -f openai-cpa-local >/dev/null 2>&1 || true) && docker build -t openai-cpa-local:latest . && docker run -d --name openai-cpa-local -p 8000:8000 -v /Users/meilinwang/Projects/openai-cpa-Public/data:/app/data -v /var/run/docker.sock:/var/run/docker.sock --add-host=host.docker.internal:host-gateway openai-cpa-local:latest
```

### 重启后恢复这套 20 并发集群的推荐顺序

#### 1. 重启主容器


#### 2. 登录并通过系统接口重新同步 `20` 个实例

```bash
cd /Users/meilinwang/Projects/openai-cpa-Public && python3 - <<'PY'
import json, urllib.request
base='http://127.0.0.1:8000'
login_req=urllib.request.Request(base+'/api/login', data=b'{"password":"admin"}', headers={'Content-Type':'application/json'}, method='POST')
login=json.load(urllib.request.urlopen(login_req, timeout=15))
token=login['token']
headers={'Content-Type':'application/json','Authorization':f'Bearer {token}'}
for path,payload in [
    ('/api/clash/deploy', {'count':20}),
    ('/api/clash/update', {'sub_url':'https://224fr.no-mad-sub.one/link/9oLEhZPmpFuHq5OU?clash=3&extend=1','target':'all'})
]:
    req=urllib.request.Request(base+path, data=json.dumps(payload).encode(), headers=headers, method='POST')
```

如果只做了“同步实例”，新创建出来的 Mihomo 实例可能只有默认骨架配置，运行时会找不到 `🔰 选择节点` 这样的策略组。

```bash
cd /Users/meilinwang/Projects/openai-cpa-Public && mkdir -p data && [ -f data/config.yaml ] && cp data/config.yaml data/config.yaml.bak-$(date +%Y%m%d-%H%M%S) || true && (docker rm -f openai-cpa-local >/dev/null 2>&1 || true) && docker build -t openai-cpa-local:latest . && docker run -d --name openai-cpa-local -p 8000:8000 -v /Users/meilinwang/Projects/openai-cpa-Public/data:/app/data -v /var/run/docker.sock:/var/run/docker.sock --add-host=host.docker.internal:host-gateway openai-cpa-local:latest
```
