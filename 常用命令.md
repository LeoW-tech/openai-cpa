# openai-cpa 常用命令

这份文档专门存放本项目的常用命令。当前项目同时有 `mac` 本地运行环境和 `Linux` 正式运行环境，所有命令都已经整理为“带真实路径、整行可复制执行”的单行命令。

## 环境速查表

| 项目 | mac 本地环境 | Linux 正式环境 |
| --- | --- | --- |
| 部署根目录 | `/Users/meilinwang/Projects/openai-cpa-Public` | `/srv/openai-cpa` |
| 仓库目录 | `/Users/meilinwang/Projects/openai-cpa-Public` | `/srv/openai-cpa/repo` |
| 数据目录 | `/Users/meilinwang/Projects/openai-cpa-Public/data` | `/srv/openai-cpa/data` |
| 配置文件 | `/Users/meilinwang/Projects/openai-cpa-Public/data/config.yaml` | `/srv/openai-cpa/data/config.yaml` |
| 容器名 | `openai-cpa-local` | `openai-cpa` |
| 主要入口 | 本地脚本 / 单容器 | `/srv/openai-cpa/docker-compose.linux.yml` |

## 使用规则

- 看到 `Linux 版` 只在 Linux 正式环境执行。
- 看到 `mac 版` 只在 mac 本地环境执行。
- 没标环境且命令完全一致时，表示两端通用。
- `mac` 本地容器脚本只适用于 `/Users/meilinwang/Projects/openai-cpa-Public`，不适用于 Linux 正式环境。
- `Linux` 正式环境命令默认针对 `/srv/openai-cpa` 或 `/srv/openai-cpa/repo`，不要在 mac 路径执行。
- 修改配置前，先确认目标数据目录，避免把 mac 的 `data/config.yaml` 和 Linux 的 `/srv/openai-cpa/data/config.yaml` 混用。

当前本机日常维护默认以根目录 `scripts/` 下的脚本和本地容器 `openai-cpa-local` 为准。
README 里的 `docker-compose.yml` 更接近上游示例，不是这里日常在 `main` 分支维护本机服务的主入口。

## 高频命令

### 1. 快速重启当前服务

Linux 版：

```bash
cd /srv/openai-cpa && docker compose -f docker-compose.linux.yml restart openai-cpa
```

Linux 版兜底：

```bash
docker restart openai-cpa
```

mac 版说明：本机日常维护默认按 `main` 分支重启当前容器。这条命令不会重新构建镜像，也不会把你刚 `git pull` 下来的新代码装进容器里。

mac 版推荐：

```bash
cd /Users/meilinwang/Projects/openai-cpa-Public && git checkout main && ./scripts/restart_local_container.sh
```

mac 版兜底：

```bash
cd /Users/meilinwang/Projects/openai-cpa-Public && git checkout main && docker restart openai-cpa-local
```

### 2. 有代码修改时重建并重新启动

Linux 版：
适用场景：你想确保容器实际运行的是 `origin/main` 最新代码，或者你已经改了项目代码、依赖、镜像构建内容，需要重建镜像再启动。
`./scripts/rebuild_local_container.sh` 会自动备份当前 `data/config.yaml`。

```bash
cd /srv/openai-cpa && docker compose -f docker-compose.linux.yml up -d --build
```

mac 版推荐：

```bash
cd /Users/meilinwang/Projects/openai-cpa-Public && git checkout main && git pull --ff-only origin main && ./scripts/rebuild_local_container.sh
```

mac 版兜底：

```bash
cd /Users/meilinwang/Projects/openai-cpa-Public && git checkout main && git pull --ff-only origin main && mkdir -p data && [ -f data/config.yaml ] && cp data/config.yaml data/config.yaml.bak-$(date +%Y%m%d-%H%M%S) || true && (docker rm -f openai-cpa-local >/dev/null 2>&1 || true) && docker build -t openai-cpa-local:latest . && docker run -d --name openai-cpa-local -p 8000:8000 -v /Users/meilinwang/Projects/openai-cpa-Public/data:/app/data -v /var/run/docker.sock:/var/run/docker.sock --add-host=host.docker.internal:host-gateway openai-cpa-local:latest
```
